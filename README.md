# calculator

## Описание
Это простая программа-калькулятор для консоли на языке C#. Она умеет делать базовые операции с числами, как сложение, вычитание, умножение, деление, остаток от деления, квадрат числа, квадратный корень, обратное число. Ещё есть простые функции с памятью: добавить в память, вычесть из памяти и показать, что в памяти. Программа работает в цикле — можно считать много раз, пока не выйдешь.

## Использование
1. Соберите и запустите программу.
2. Следуйте подсказкам:
   - Введите первое число.
   - Введите действие (например, +, -, *, /, %, sqr, sqrt, inv, m+, m-, mr).
   - Если действие два числа (как +, -, *, /, %), введите второе число.
   - Для действий с одним числом (sqr, sqrt, inv) или с памятью второе число не нужно (если программа спросит, оно просто не используется).
3. Посмотрите на результат.
4. Нажмите y, чтобы продолжить, или n, чтобы выйти.

Что значат действия:
- `+`: Сложить числа
- `-`: Вычесть числа
- `*`: Умножить числа
- `/`: Поделить числа
- `%`: Остаток от деления
- `sqr`: Квадрат первого числа (x^2)
- `sqrt`: Квадратный корень первого числа
- `inv`: Обратное первого числа (1/x)
- `m+`: Добавить первое число в память
- `m-`: Вычесть первое число из памяти
- `mr`: Показать число из памяти

## Ограничения
Программа не даёт делать неправильные вещи, чтобы избежать ошибок:
1. **Деление на ноль**: Нельзя делить или брать остаток на ноль. Программа скажет об ошибке и не посчитает.
2. **Корень из минуса**: Нельзя брать квадратный корень от отрицательного числа. Будет ошибка.
3. **Обратное от нуля**: Нельзя брать обратное (1/x) от нуля. Будет ошибка.
4. **Плохой ввод числа**: Числа должны быть нормальными (с плавающей точкой). Если нет, программа попросит ввести заново.
5. **Плохое действие**: Только те действия, что в списке. Другие — ошибка.

Это сделано, чтобы программа работала без сбоев.

## Известные проблемы
- **Нет объяснений в коде**: В коде нет комментариев, из-за чего другим людям сложно понять, как всё устроено, особенно где switch и проверка ввода.
- **Всё в одном месте — в Main**: Нет отдельных функций для разных частей (например, для расчётов или ввода). Из-за этого код трудно менять или проверять.
- **Использование double вместо float**: Double точнее, но для очень маленьких или больших чисел может дать неожиданные результаты.
- **Нет защиты от слишком больших чисел**: Если умножить огромные числа, программа может сломаться из-за переполнения, без специальной проверки.
- **Программа ломается при неверном знаке продолжения**: Если ввести символ отличный от y или n, программа завершится с ошибкой.
